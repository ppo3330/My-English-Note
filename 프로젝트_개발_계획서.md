# My English Note 2주 개발 계획서

## 1. 계획서 목적

이 문서는 My English Note 웹사이트를 어떤 순서로 만들지 정리한 개발 계획서다.

기획서가 “무엇을 만들지”를 정리한다면, 계획서는 “언제, 어떤 순서로 만들지”를 정리한다.

---

## 2. 2주 전체 목표

2주 동안 다음 두 단계를 진행한다.

```text
1주차: 프론트엔드 첫 버전 완성
  → HTML, CSS, JavaScript, localStorage
  → 내 브라우저에서 작동하는 영어 학습 웹사이트

2주차: 백엔드와 DB 추가
  → Node.js 서버, API, 데이터베이스
  → 데이터를 브라우저가 아니라 서버 DB에 저장하는 웹사이트
```

1주차의 목표는 “눈에 보이고 직접 사용할 수 있는 첫 버전”을 완성하는 것이다.

2주차의 목표는 “실제 웹서비스 구조에 가깝게 서버와 DB를 붙여보는 것”이다.

## 3. 사용할 기술

### 1주차 기술

- HTML
- CSS
- JavaScript
- `localStorage`
- Vite
- Git

### 2주차 기술

- Node.js
- Express
- SQLite 또는 PostgreSQL
- Prisma 또는 SQL 기초
- REST API
- `fetch`
- 환경 변수

처음 백엔드를 붙일 때는 SQLite가 더 쉽다. 실제 서비스형 DB까지 경험하고 싶다면 PostgreSQL을 사용한다.

### 1주차에 만들 기능

- 단어, 구동사, 문장 세 노트
- 항목 추가
- 항목 목록 보기
- 항목 수정

- 검색
- 브라우저 저장

- 직접 입력 시험

### 2주차에 추가할 기능

## 6. 1주차 개발 일정 — 프론트엔드 첫 버전 완성

### Day 1 — 기본 화면과 입력 흐름

목표:

- HTML, CSS, JavaScript 연결하기
- 노트 선택 버튼 만들기
- 영어와 뜻 입력창 만들기
- 저장 버튼을 누르면 화면 목록에 표시하기

완료 기준:

- 단어와 뜻을 입력하면 현재 노트 목록에 보인다.

### Day 2 — 세 노트와 데이터 구조

목표:

- 단어, 구동사, 문장 노트 구분하기
- 항목을 객체 형태로 저장하기
- 배열로 여러 항목 관리하기

완료 기준:

- 세 노트의 데이터가 서로 섞이지 않는다.

### Day 3 — 추가, 수정, 삭제, 검색

목표:

- 항목 추가 기능 정리
- 수정 기능 만들기
- 삭제 기능 만들기
- 영어 또는 한국어 검색 만들기

완료 기준:

- 항목을 추가, 수정, 삭제, 검색할 수 있다.

### Day 4 — localStorage 저장

목표:

- 데이터를 브라우저 `localStorage`에 저장하기
- 새로고침 후 다시 불러오기
- 저장 관련 코드를 `storage.js`로 분리하기

완료 기준:

- 새로고침해도 저장한 항목이 남아 있다.

### Day 5 — 카드 학습

목표:

- 저장된 항목을 카드로 보여주기
- 카드를 누르면 정답 보이기
- 영어→뜻, 뜻→영어 방향 선택하기
- 알고 있음, 헷갈림, 모름 상태 표시하기

완료 기준:

- 저장된 항목으로 카드 학습을 할 수 있다.

### Day 6 — 직접 입력 시험

목표:

- 랜덤 문제 만들기
- 답 입력창 만들기
- 정답 확인하기
- 시험 결과와 오답 목록 보여주기

완료 기준:

- 직접 입력 시험을 보고 결과를 확인할 수 있다.

### Day 7 — 1차 버전 정리와 배포

목표:

- 전체 기능 테스트하기
- 모바일 화면 확인하기
- 오류 수정하기
- 정적 웹사이트로 배포하기
- 1주차 학습 내용 정리하기

완료 기준:

- 배포된 주소에서 프론트엔드 첫 버전을 사용할 수 있다.

## 7. 2주차 개발 일정 — 백엔드와 DB 추가

### Day 8 — 백엔드 서버 시작

목표:

- `server/` 폴더 만들기
- Node.js와 Express 서버 만들기
- `/health` API 만들기
- 프론트엔드와 백엔드를 함께 실행하는 방법 정리하기

완료 기준:

- 브라우저에서 백엔드 API 응답을 확인할 수 있다.

### Day 9 — DB 연결과 테이블 설계

목표:

- SQLite 또는 PostgreSQL 선택하기
- 학습 항목 테이블 설계하기
- DB 연결 코드 만들기
- 테스트용 항목을 DB에 저장해보기

완료 기준:

- DB에 학습 항목 데이터를 저장할 수 있다.

### Day 10 — 항목 API 만들기

목표:

- 항목 목록 조회 API 만들기
- 항목 추가 API 만들기
- 항목 수정 API 만들기
- 항목 삭제 API 만들기

완료 기준:

- API만으로 항목 CRUD가 가능하다.

### Day 11 — 프론트엔드와 API 연결

목표:

- 기존 `localStorage` 저장 코드를 API 호출 코드로 바꾸기
- `fetch`로 항목 조회, 추가, 수정, 삭제하기
- 로딩 상태와 오류 메시지 처리하기

완료 기준:

- 프론트엔드에서 입력한 항목이 DB에 저장된다.

### Day 12 — 카드 학습과 시험을 DB 데이터로 연결

목표:

- 카드 학습이 서버에서 불러온 데이터로 작동하게 하기
- 직접 입력 시험이 서버 데이터로 작동하게 하기
- 학습 상태를 DB에 저장할지 범위 결정하기

완료 기준:

- DB에 저장된 항목으로 카드 학습과 시험을 할 수 있다.

### Day 13 — 백엔드 예외 처리와 정리

목표:

- 잘못된 입력 처리하기
- 없는 항목 수정/삭제 요청 처리하기
- 서버 에러 메시지 정리하기
- 프론트엔드 저장 코드와 서버 코드를 정리하기

완료 기준:

- 주요 오류 상황에서 웹사이트가 멈추지 않는다.

### Day 14 — 최종 테스트와 문서화

목표:

- 프론트엔드와 백엔드 전체 흐름 테스트하기
- 서버 실행 방법 작성하기
- DB 사용 방법 작성하기
- 2주 학습 내용 정리하기

완료 기준:

- 로컬에서 프론트엔드, 백엔드, DB를 실행해 전체 기능을 사용할 수 있다.

## 10. 2주 완료 기준

2주 프로젝트는 다음 조건을 만족하면 완료다.

### 1주차 완료 기준

- 세 노트를 사용할 수 있다.
- 항목을 추가, 수정, 삭제, 검색할 수 있다.
- 데이터가 `localStorage`에 저장된다.
- 카드 학습을 할 수 있다.
- 직접 입력 시험을 할 수 있다.
- 모바일과 PC에서 사용할 수 있다.
- 정적 웹사이트로 배포되어 있다.

### 2주차 완료 기준

- 백엔드 서버가 실행된다.
- DB가 연결된다.
- 항목 데이터가 DB에 저장된다.
- 프론트엔드가 API를 통해 데이터를 가져온다.
- 카드 학습과 시험이 DB 데이터로 작동한다.
- 서버와 DB 실행 방법이 문서화되어 있다.

## 11. 2주 이후 확장 계획

2주 안에는 백엔드와 DB의 기본 흐름까지만 만든다. 그 다음 단계에서 실제 서비스 기능을 추가한다.

추천 순서:

1. 회원가입 만들기
2. 로그인 만들기
3. 사용자별 데이터 분리하기
4. 여러 기기에서 같은 데이터 사용하기
5. 서버와 DB 배포하기
6. 백업과 복원 기능 다시 설계하기
7. React 또는 TypeScript로 프론트엔드 구조 개선하기
8. 반복 학습 알고리즘 추가하기

첫 2주의 핵심은 완벽한 서비스를 만드는 것이 아니라, **프론트엔드 → 백엔드 → DB로 이어지는 웹 개발의 전체 흐름을 직접 경험하는 것**이다.
